Shade Library Preferences

To display the current print preferences select menu option Edit->Preferences then click on the Shade Library tab.

 

An example of the Shade Library tab in Preferences

 

 

Database

You can use the data base directly, or, your can use a separate text file - a shadefile. Taking the former option avoids the need to "export shade file" from C+Y. It is thus recommended that you select "Use C+Y database direct". * (See below for more information)

Replace Missing Shades

In order to aid setting up a new C+Y data base, this option was introduced. It forces the software to write shades found in a design file to the C+Y database if they are "missing". At present this applies only to .DES design files.

Do not check palette against the shade file

If selected the program will not update shade information in the palette on opening a design. Existing shade names and RGB values in the palette will be retained.

Discard conflicting shades

If selected the program will remove any shade numbers from the palette where there is a conflict with the shade file. A conflict is any shade now missing from the shade file or where the RGB value is different. Note that only the shade numbers and names are removed from the palette. The color itself will be retained.

Resolve differences

If selected the program will resolve any differences between the design palette and the shade file by using one of the following methods.

Always trust the shade library

If selected, then in the event of a conflict, the RGB value from the shade library will be used automatically.

Always trust the design file

If selected, then in the event of a conflict, the RGB value from the design file will be used automatically.

Always ask the user

If selected, then in the event of a conflict, a dialog will be displayed allowing the user to choose whether to use the the RGB value from the design file or the shade file (see below)

 

* Using C+Y Database Direct

Select this option in order to get plain shades from the Colors and Yarns database direct without going through an external "shade file". This should be a simpler option for most users as it does not involve "exporting" a shade file from C + Y and it is recommended that all users select this option as soon as this version of the software is installed.

Note that after an update to the Colors and Yarns database you should select the button "Update database for other applications" in C + Y. This will send a message to the design program (and other APSO software) instructing it to update designs and shade lists. The update will include designs and palettes in print layouts in all projects.

Note that if the software cannot open the database you will be offered the option of trying to load the text file version.

(Applies only to plain shades at present).

 

Resolving Differences

If you have selected "Resolve Differences" together with "Always Ask the User" then the following dialog will be displayed if a conflict arises when opening a design file.

Normally there will be two lines shown, the second line being taken from the shade file. In the unlikely even that there are two or more versions of the shade in the shade file, they will both (or all) be listed.

Example of the Select Shade dialog. Notice that the second line shows a slightly different RGB value meaning that the shade file has been changed since the design was last saved. The user has selected that line so that the new RGB value will be substituted.

 

Colour editing Preferences

The software allows you to edit RGB values in a design, even if that RGB value has been taken from the shade file. This preference allows you to specify whether or not to retain the shade number and description in that case.

Clear the shade code and description

If an RGB value is edited when a shade code is present then the shade code and description are cleared.

Preserve the shade code and Description

If an RGB value is edited when a shade code is present then the shade code and description are retained.

 

The second of these options will cause a conflict if in fact a change is made to a shade in the design and it is then saved and subsequently reloaded.
