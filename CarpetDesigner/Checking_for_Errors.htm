Checking for Frame Plan Errors

 

The procedure for checking for errors really depends on the setting of the Automatic Refresh flag. If you have set this flag then the frame plan is updated automatically every time you edit the design. If it is not then you must correct any errors manually.

Errors are colours missing from a frame plan in a particular end. The software will not report an error if a colour is not in the correct frame according to the custom planting dialog or the specified frame order.

If a frame plan is locked, then errors are not automatically corrected regardless of the preference setting.

Quick Check

The easiest and quickest way of checking the frame plan is to select the menu item Frames->Check. This will display a message indicating whether there are any errors or not. If the Frame plan is not currently being viewed, it will be switched into view mode.

An error will be reported if any of the following are true:

  • There are colours in error frames.

  • One or more ends in the design contains a colour which is not present in the corresponding end in the frame plan.

If a colour is present in a position which is not actually required, this will not be reported as an error. This means that you can have a single frame plan which is used by several designs. The frame plan can be loaded (see the topic Loading Frames and Locking Frames) and checked against each design.

Although you can check frames visually (see below) it is worth getting into the habit of doing this check as described here before saving a design as a confirmation that the frame plan is correct.

 

Visual Check

If you have created a frame plan or if you have selected the Automatic refresh option and the Frame plan is Unlocked, then the frame plan will have been updated each time a design edit is done and so you can check visually for error frames at any time.

Example of a section of a frame plan showing error frames (outlined in red). Since the 8 actual frames are full, the red and pink colours must be removed from the design in this area as they cannot be planted. Alternatively of course, other colours could be removed from the design to make space for the red and pink.

Error frames indicate where a colour is used in the design in an end but is not yet placed into a frame. Error frames will always contain 1 colour per "frame". If you see 3 error frames then there are 3 colours for which errors exist.

 

Locked Frame Plan Checks

Select Frames->Frame Locking->Lock Frames in order to prevent the software automatically updating the frame plan and to enable the following two options.

  • Show Design Errors - Select this option to mark all points in the design where a colour is used that is not present in the frame plan. This may be switched on and off using the same menu item. The mark consists of diagonal yellow lines, similar to the protected areas display.

  • Show Unused Frames - Select this option to mark points in the frame plan at which a colour has been placed but which is not actually required for this design. (Each "point" in the frame plan represents a peg; or the bobbin colour if present). Again, an unused bobbin will be marked by diagonal yellow lines. This feature may be switched off by selecting the same menu item.

Example of the "Show Design Errors" and "Show Unused Frames" options. Note the 3 marked points in the frame plan indicating unused bobbins and the marked area in the design which does not have corresponding blue bobbins in the frame plan.

Note that you can easily move forwards and backwards through the design errors by pressing * and / on the numeric keypad. The software will bring each design point in error to the centre of the design window going down each end presenting each separate block of errors in turn and working from left to right. This is particularly helpful if there are just one or two small errors in a large design, as it is not easy to see the errors otherwise.

Remove errors by normal design changes or by editing the frame plan (see the topic Frame Edit).
