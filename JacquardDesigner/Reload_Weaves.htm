Reload Weaves

This operation forces the software to reload all weaves in the schema.

It is found on the pop up menu in Apply Weaves (right click on the schema), or you can reload an individual weave using the Reload Weave option on the weave edit menu (right click on the weave image in Apply Weaves).

It should be used when you wish to refresh the weaves after the weave files have been edited or possibly moved to a new path location (see below).

The software does not remove any of the stitching, manual edits, shift, inverse or flip flags.

Weaves in Design Files

From version 5.2.2.1 onwards, weaves are held in the design file (iff or prj), so strictly speaking it should never be necessary to reload the weaves for designs created with this or a later version. The fact that copies of the weaves are held within iff files makes it much easier to move a design file from one machine to another on which weaves may be in different locations.

Weave Path and Design Files

When Reload Weaves is used, if a weave cannot be found in its original location, the software will search the default location given in preferences and will update the weave path for each weave in the schema if necessary.
