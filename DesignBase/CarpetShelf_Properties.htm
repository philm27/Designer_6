Shelf Properties

 

To display the Shelf properties select menu option Shelf->Properties, or select Properties on the Shelf Pop-up menu.

 

Shelf Properties Tab

Shelf Size

You can if you wish have a total shelf size of 1000 items.

Display

This specifies the height in pixels of each shelf item. A good figure to use is between 60 and 120 pixels. When using the shelf you can scroll it to view all items.

Set these values as defaults...

If you check this box then the figures used will become the default values for all new designs when you close the dialog using the OK button.

Text

On some monitors with very high resolution, the shelf numbers can be rather small - modify the font size and/or type as necessary. A value in the range 7-12 is normally sufficient. You can click on the "Browse" ("...") button to select a font and point size.

 

Quality Tab

Ask on Import

If this is checked  then, when images are imported to the shelf and they have a different quality to the main design, the software will ask the user what to do.

Always do this

As an alternative to the above, the software will take the specified action without asking the user, (selected from the drop down list). The choices being:

  • Redraft using Smart Draw - i.e. make it the same quality as main design by re-drafting

  • Normal Redraft - same as above but use normal re-draft technique rather than Smart Draw

  • Simply Modify Figures - re-set the quality to the same as the main design which will in turn change the size

  • Leave as it is - I'll modify later

 

 

Shade Matching Tab

These setting again relate to images imported to the shelf and deal with the rules for how the software will match to existing palette entries. This is only relevant when the user clicks on "No" in the following message when importing images to the shelf.

 

Always match to identical shade code

If this is checked the software will try to match shade codes in incoming palettes to shade codes in existing palette.

RGB Values

You can specify a tolerance within which the software will assume the RGB value is the same. For example if the Red Green and Blue values in an incoming shade are within 5 of an RGB value in the existing palette, then the software will assume they are the same and will use the existing palette index for the incoming shade.
