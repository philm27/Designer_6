Random Balance

This pair of functions (Horizontal and Vertical) allow the user to force a design to be balanced in one or other axis.

If a design is nearly balanced anyway it will be very quick but will be slower if a design is very unbalanced.

A Vertical Random Balance will make each END in the design contain the same amount of color as every other end (or as near as is possible). Any colors that occur less than once in every end (e.g. 24 times in a width of 252) will be evenly spread across the design.

Similarly a Horizontal Random Balance will make each ROW in the design contain the same amount of color as every other row.

These functions will not move whole shapes around in order to balance a design. Instead, individual pixels will be moved in order to balance the image, thus an image that is very "unbalanced" will be substantially modified by this process.

Stencil and Protected Areas

Note that these operations do not use the current stencil and Protected areas mask and will always act on the full design.
