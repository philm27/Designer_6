File Preferences

 

 

Same for Load and Save

If this is checked, there will be a different default directory for load operations and save operations. The default directory is reset each time a particular directory is accessed.

For more information see the topic Default Directories.

Clear filename on Save AS

If this is checked, the file name will initially be cleared on using the "Save As" option. This forces the user to type in the new name each time.

PAT file options

Save design with no zero colour - if checked forces the software to save PAT files with the first colour in the palette effectively deleted. If "Save design using the first..." is selected, then the first palette colour is the "zero" colour and is saved as colour zero in the file. By convention the "zero" colour is transparent in some designs.

These options are mutually exclusive of course.

For more information see the topic Pat File Options.

Bitmap File Options

The "Additional Information" option causes BMP files to be created that can be read by programs such as the Calidus room layout program. This option does not generally mean that the files cannot be read by other programs therefore the option can remain selected on a permanent basis.

These options are mutually exclusive of course.

KNF/DNF file options

When using certain loom types there can sometimes be a problem with the palette entry numbers. Normally the software will save the first palette entry as colour number zero. If this causes problems, this option "KNF/DNF file data is 1 based" may be useful.

With this option checked, the first palette colour will be numbered 1 in the output file.

IFF Embedded Thumbnail

When saving an IFF file, the software can be made to store a JPEG image within the IFF file, this will be used (subject to correct drivers being installed) by VISTA (tm) and later operating systems to display thumbnails in file explorer and directory lists.

The embedded thumbnail is also used by APSO designer programs in the file open preview in order to speed up the display of large designs.

It is therefore recommended that the "Enabled" box is checked and that the quality is set to a value in the range 60-70. This gives a reasonable balance between additional file size and quality of image. This setting does not in any way affect the actual design image when loaded - it only affects the thumbnail shown in the File Open dialog and possibly in File Explorer and directory lists.

As indicated in the dialog itself, a high value such as 99 will make the file larger but will give a better quality image. A very low value will give a very poor quality thumbnail but a very small file. However, values below 10 are not recommended.

Note also that this feature only applies to IFF files.
