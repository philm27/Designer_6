Zero Color

This feature is intended to be used in designs in which there are areas where no yarn is to be used (in Axminster) or all yarns are woven in the back (in Wilton or Pile Fabric).

Only the first palette colour can be flagged as a "Zero Color" and once set it cannot be moved to another position in the palette though it can be "un-flagged" using the menu option (see below).

The "Zero Color" will be marked in the palette as shown here by a small "Z" motif.

The colour can be used normally but all areas in which this colour is used are assumed to be "no yarn" areas. These areas can optionally be excluded from the Usage% figures. They will automatically be excluded from frame plan.

Specifying the zero color

If there isn't currently a zero color, you can create one in two ways by selecting one of the Palette menu options...

Number from Zero will simply cause palette numbering to start at zero instead of 1. Thus what was previously color #1 will become col #0, what was #2 will become #1 etc. This is purely a cosmetic i.e. labelling change - there is no change to the design or palette.

Insert Zero Color will cause a new, therefor unused,  zero color to be inserted at front of the palette.

Removing Zero Color

If there is currently a Zero color, i.e. both options currently checked, you can remove it by selecting one of the same two menu options...

Number from Zero to simply cause numbering to start at 1 leaving the palette intact

Insert Zero Color to remove the first color in the palette. In this case the software will check to make sure the color is not used. If it is used you will have to remove it from the design before re-trying this option.

Palette Menu

The two menu options (see below) will be checked if a there is a zero color in the palette.

BIN file and Van de Wiele export

When exporting files of these two types directly from Carpet Designer, any "Zero Color" areas will be assumed to be "backing only" and the appropriate weaves will be used in those areas.

Changing the Zero Color

The RGB value of the Zero colour can be modified in the normal way as described in the topic Palette Change.

The user can also select a shade from the Shade Library or Colors and Yarns data base for the Zero Color and this could be used to give it not only a standard RGB value but also a standard name in printouts and on the Palette Information window.

The shade code could, for example, be "FELLING" or whatever else is required.
