Export - Format Files

 

Introduction

The Export feature uses format files to control the file content of the exported patterns.

These give the software important information such as the number of frames, the number of holes in the card or "points" in the data per frame, the number of picks (cards) per design row and the actual positions of each end and frame in the card as well as a standard weave. Details of the content of the format file is given below.

The format file is a simple text file with a .TXT file extension so that it can be edited easily using Notepad or a similar text editor.

The initial set of files will normally be supplied by APSO and an example installation follows:

The above assumes that the main drive is c: drive. If this is not the case simply change the drive letter to d: or e: as appropriate.

If you wish to place these files elsewhere then that is no problem as the system allows the user to browse to a different directory and remembers where they are located after they have been used.

 

Format File Content

The file consists of any number of header lines followed by any number of detail lines.

Comments can also be included in any line by using two slashes "//". Anything on a line after this is ignored by the system.

The general form of the file is therefore:

Header1

.

Header n

Detail 1

Detail 2

.

Detail n

 

Completely blank lines can be inserted anywhere. Lines containing only a comment are allowed and a comment can be added to the end of any line.

Tab characters can be used between words or values in order to make the file more readable.

 

Header Lines

A header line consists of any number of statements separated by commas. Each statement has the following form:

Keyword = value

 

For example:

SIZE = 1344, FRAMES = 3

 

Statements can be in any order and there can be any number of them on one line.

 

Keywords

Keyword

Value

Default Value

Description

SIZE

33 - 9999

1344

Card Size. number of holes on card or hooks in file

SETS

1 - 8

1

Specifies the number of sets required across the loom. If a 12 foot wide loom uses Verdol papers and each set covers a 2 foot width of carpet then there would be 6 sets. The software assumes that all sets have the same layout. A file will automatically be exported for each set.

FRAMES

2 - 12

1

Number of Frames

TYPE

VERDOL or WEAVE

VERDOL

Specifies card or file type. Only VERDOL is supported at present. WEAVE allows a simple file to be created mainly for testing purposes. May be omitted.

CARDSPERROW

1 or 2

1

Specifies number of cards per design row.

HOLESPER

1 or 2

1

Specifies the number of holes per yarn i.e. per frame within a dent. 3 position wilton looms normally require 2 holes in a card in order to control all 3 shed heights

SEL1

0, 1, 00, 01, 10, or 11

1

Specifies the punch pattern for a single yarn WHEN SELECTED. If there is only 1 hole per yarn then the value must be 0 or 1 (1 being a punched hole). If there are 2 holes per yarn this must be 00, 10, 01, or 11. If there are two cards per design row this relates only to the first of the two cards. These punching pattern keywords (SEL1 UNSEL1 etc) are only used by CarpetDesigner and enable it to construct the weaves.

UNSEL1

0, 1, 00, 01, 10, or 11

0

Specifies the punch pattern for a single yarn WHEN NOT SELECTED. It must be a 1 or 2 digit number as for SEL1 above and is only used by Carpet Designer not by Jacquard Designer.

SEL2

0, 1, 00, 01, 10, or 11

1

This is used in the same way as SEL1 above but applies to the second of the two cards when CARDSPERROW is 2.

UNSEL2

0, 1, 00, 01, 10, or 11

0

This is used in the same way as UNSEL1 above but applies to the second of the two cards when CARDSPERROW is 2.

SEL

0, 1, 00, 01, 10, or 11

1

May be used instead of SEL1 if only 1 card - see above.

 

UNSEL

0, 1, 00, 01, 10, or 11

0

May be used instead of UNSEL1 if only 1 card - see above.

MINCARDS

0-999

0

Minimum number of cards required. Additional repeats will be added to the file if the design repeat is less than this figure.

HEADER

1 - 10

0

Number of header cards to be punched (verdol papers only). The First header card contains the design name and set A, B, C etc in the centre, with a fully punched edge at left and right. The second header card, if punched, contains the card count. The third header card onwards, if punched, will have every frame selected so that these cards can be made to overlap the last few cards and glued to make a card loop. (Verdol papers only).

CARDNUMBERS

YES or NO

NO

Whether or not to punch card numbers into unused space in the card. The software will automatically find space for up to 4 digits. A minimum of 3 card columns is required for each digit, though the program will use 4 columns if available. (Verdol papers only).

 

Detail Lines

If detail lines are not supplied then the software will assume that the data is to be written in normal sequence i.e. dent 1 frame 1, dent 1 frame 2 dent 1 frame 3 etc etc.

Some looms require that the data is arranged differently particularly if the output is intended for punching on Verdol papers and the detail lines allow each pick to be written in any order.

Detail lines have two possible formats depending on the number of holes per yarn (HOLESPER value from the header).

For 1 hole per yarn:

Hole, end, frame // comment

For 2 holes per yarn:

Hole1, hole2, end, frame // comment

In both cases the comment is optional.

The sequence of the lines does not matter.

The Hole number (or hook number) for Verdol papers will be in the range 1 - 1344. Hole 1 being at the top left of the card, hole 2 being the one below it in the same column. Hole 8 is therefore at the bottom of the first column and hole 9 at the top of the second column.

The software will check that a valid amount of data is present i.e. for any 3 frame loom, the number of detail lines must be a multiple of 3. Also, each of the 3 frames must be specified once and once only for each dent. Similarly each hole can only be used once though, of course, not every hole will be used.

 

3 Frame Verdol Paper Example

The Format File for a 3 frame Verdol paper loom might have content as follows:

SIZE = 1344, SETS = 6, FRAMES = 3 ,

HOLESPER = 2, CARDSPERROW = 2,

SEL1 = 00, SEL2 = 11, // when selected card 1 is 00 card 2 is 11

UNSEL1 = 00, UNSEL2 = 10 // when NOT selected card 1 is 00 card 2 is 10

HEADER = 2, CARDNUMBERS = YES // a 2 card header and do card numbers

MINCARDS = 48 // minimum cards 48

  53, 54, 1, 3// Holes 53 and 54 are end 1 frame 3

  57, 58, 1, 2// Holes 57 and 58 are end 1 frame 2

  63, 64, 1, 1// Holes 63 and 64 are end 1 frame 1

  65, 66, 2, 3// Holes 65 and 66 are end 2 frame 3

  67, 68, 3, 3// Holes 67 and 68 are end 3 frame 3

  69, 70, 4, 3// Holes 69 and 70 are end 4 frame 3

  71, 72, 2, 2// Holes 71 and 72 are end 2 frame 2

  73, 74, 3, 2// etc etc

  75, 76, 4, 2

  77, 78, 2, 1

  79, 80, 3, 1

  81, 82, 5, 3

  83, 84, 6, 3

  87, 88, 5, 2

  89, 90, 6, 2

  91, 92, 4, 1

  93, 94, 5, 1

  95, 96, 6, 1

  97, 98, 7, 3

  99, 100, 8, 3

 101, 102, 7, 2

 103, 104, 8, 2

 105, 106, 9, 2

 

Note not all detail lines shown.

 

 

5 frame Verdol Paper Example

A 5 frame Verdol paper format might be as follows (1 hole per yarn and 1 card per row):

SIZE = 1344, SETS = 6, FRAMES = 5

HEADER = 2, CARDNUMBERS = YES // a 2 card header and do card numbers

MINCARDS = 48 // minimum cards 48

  66, 1, 5 // Hole 66 is end 1 frame 5

  69, 1, 4 // Hole 69 is end 1 frame 4

  72, 1, 3 // Hole 72 is end 1 frame 3

  75, 1, 2 // Hole 75 is end 1 frame 2

  78, 1, 1 // Hole 78 is end 1 frame 1

  82, 4, 5 // Hole 82 is end 4 frame 5

  83, 3, 5 // Hole 83 is end 3 frame 5

  84, 2, 5 // Hole 84 is end 2 frame 5

  85, 4, 4 // Hole 85 is end 4 frame 4

  86, 3, 4 // Hole 86 is end 3 frame 4

  87, 2, 4 // Hole 87 is end 2 frame 4

  88, 4, 3 // Hole 88 is end 4 frame 3

  89, 3, 3 // Hole 89 is end 3 frame 3

  90, 2, 3 // Hole 90 is end 2 frame 3

 

Note not all detail lines shown.

 

 

 

Van de Wiele Electronic Jacquard 3 Frame 12ft NTTB example

This file and other formats for the Van De Wiele Looms do not need complicated "Harness" information as the sequence is normal therefore the files are much shorter and the whole file is shown here. This format is for a 12 foot loom at 8 per inch so 144 inches x 8 per inch x 3 frame = 3456 which gives the size of the format

SIZE = 3456, FRAMES = 3 , HOLESPER = 1, CARDSPERROW = 2,

SEL1 = 0, SEL2 = 1, // when selected card 1 is 0 card 2 is 1

UNSEL1 = 0, UNSEL2 = 0 // when NOT selected card 1 is 0 card 2 is 0

// No need for Harness info as sequence is normal straight i.e. hook 1 is end 1 frame

// 1 hook 2 is end 1 frame 2 etc etc

 

 

Van de Wiele Electronic Jacquard 3 Frame 12ft TTB - SW example

Again, there is no need for detail records and the whole file is shown here. In fact, since the weaves are identical to those for NTTB, this file is the same as that for the 12 ft NTTB above.

SIZE = 3456, FRAMES = 3 , HOLESPER = 1, CARDSPERROW = 2,

SEL1 = 0, SEL2 = 1, // when selected card 1 is 0 card 2 is 1

UNSEL1 = 0, UNSEL2 = 0 // when NOT selected card 1 is 0 card 2 is 0

// No need for Harness info as sequence is normal straight

// i.e. hook 1 is end 1 frame 1 hook 2 is end 1 frame 2 etc etc

 

 

VDW Electronic Jacquard 3 Frame 12ft TTB - TW example

As above, this file is quite short and the whole file is shown here. The only difference between this file and the previous ones is that the standard weave is different. The selected yarn is the same but when not selected the first pick is a lift and the second is a drop.

SIZE = 3456, FRAMES = 3, HOLESPER = 1, CARDSPERROW = 2,

SEL1 = 0, SEL2 = 1, // when selected card 1 is 0 card 2 is 1

UNSEL1 = 1, UNSEL2 = 0 // when NOT selected card 1 is 1 card 2 is 0

// NO need for Harness info as sequence is normal straight

// i.e. hook 1 is end 1 frame 1 hook 2 is end 1 frame 2 etc etc
